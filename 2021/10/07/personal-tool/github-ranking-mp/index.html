利用微信公众号来记录github的ranking

大概在19年左右，学习python的时候，写了一个利用selenium来写公众号文章的工具，主要是写的把github的trending页面然后排版写入到了公众号，然后自动保存和利用邮件简易的实现的公众号文章的一键发布。最近又把该小工具用java从写了一番，该文是记录重新写的过程，谨当流水账记录。
同时项目也开源到了个人的github,见链接 mp-github-trending

工具介绍

  • 抓取github的trending
  • springboot自带的定时调度
  • selenium-java实现模拟操作web动作

开发中的流水记录

  • 无脑敲代码阶段
  • 发现更好的工具阶段
  • 优化代码阶段
  • 部署服务器阶段

无脑敲代码阶段

该阶段主要就是把以前python的代码用java来实现一遍，占用了开发的大部分时间，而且开发调试也花费了很多时间。

工作内容有：
1、解决扫码发送邮件登陆问题，裁剪图片功能
2、切换页面问题
3、解决写入文章问题，富文本编辑器写入文章，利用执行js函数，然后写入
4、解决插入微信公众后的封面图，费了些时间，然后调试了半天
    4.1 xpath匹配
    4.2 利用js函数去控制css样式
    4.3 最后利用katalon recorder录入动作【完美解决】，同时其他的浏览器操作也可利用该工具，自动生成的代码比自己写的代码看起来优美。
5、点击确认群发按钮，然后发送邮件

发现更好的工具阶段

该阶段由于利用selenium去调试程序会花费大量的时间，然后也是逐渐调试逐渐总结。若后面再到类型项目的开发，便可用以下方式来加快调试开发的流程：

1、调试执行js函数的话，可先在chrome的js控制台先把函数执行完成，然后效果达到，把达到效果的js代码直接粘贴
2、强烈推荐！！！利用录制动作生成代码的工具可加快流程，如katalon recorder。

优化代码阶段

该阶段，开发工作快要接近尾声了，同时发现了利用录制动作工具也可生成代码，切生成的代码比自己用xpath挨个动作匹配生成的代码的可读性与观赏性更好。边把之前写的工具类封装代码做了一层优化。

部署服务器阶段

这一阶段，本地调试开发的流程已经完成，就是把打成的jar包扔服务器上面。大概花了一天左右的时间，面想搜索引擎最后都把问题解决了。记录遇到的问题和解决方案。

遇到的问题:
1、linux安装chrome去chromedriver驱动后、启动程序报错
2、字体显示乱码
3、click事件点击不了

解决的问题：
1、启动指定chrome和chromedriver路径
2、安装字体
3、沙盒模式启动，同时指定缩放比例（解决click点击不了）

总结经验

  • katalon recorder录制脚本工具真的好用，点赞！👍
  • linux安装chrome的selenium踩了一些坑，多试，以及观察现象，打印程序的log
